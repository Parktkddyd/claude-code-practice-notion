# 기능 단위별 커밋 도우미

## 개요

이 도구는 작업한 파일들을 **기능 단위별로 쪼개서** 단계별 커밋을 진행하도록 도와줍니다. 커밋메세지는 **한글**로 작성하고, 그 외 형식은 범용적인 형식으로 일관성 있게 유지합니다.

## 위치

```
.claude/
├── commands/
│   └── git-commit          # 터미널에서 직접 실행하는 대화형 스크립트
└── skills/
    └── git-commit/
        └── SKILL.md         # Claude Code 스킬 (/git-commit)
```

## 사용 방법

### 1. 대화형 모드 (권장)

```bash
./.claude/commands/git-commit
```

또는 스킬로:

```
/git-commit
```

**프로세스:**
1. 변경된 파일 목록이 표시됩니다
2. 스테이징할 파일을 번호로 선택합니다 (예: `1,2,3` 또는 `all`)
3. 커밋메세지를 입력합니다 (한글)
4. 커밋이 완료되면 다음 기능을 커밋할지 선택합니다
5. 모든 변경사항이 커밋될 때까지 반복합니다

### 2. 모든 변경사항 한 번에 커밋

```bash
./.claude/commands/git-commit --all
```

```
/git-commit --all
```

- 모든 변경사항을 스테이징합니다
- 커밋메세지를 입력합니다
- 바로 커밋됩니다

### 3. 스테이징된 파일만 커밋

```bash
./.claude/commands/git-commit --staged
```

```
/git-commit --staged
```

- 이미 git add된 파일만 커밋합니다
- 커밋메세지를 입력합니다

### 4. 도움말 보기

```bash
./.claude/commands/git-commit --help
```

```
/git-commit --help
```

## 커밋메세지 형식

### 기본 형식

```
<타입>: <설명>
```

### 지원하는 타입

| 타입 | 설명 | 예시 |
|------|------|------|
| `feat` | 새 기능 추가 | `feat: 로그인 폼 유효성 검사 추가` |
| `fix` | 버그 수정 | `fix: 로그인 버튼 클릭 오류 수정` |
| `refactor` | 코드 리팩토링 | `refactor: 인증 로직 최적화` |
| `style` | 코드 스타일 변경 | `style: 들여쓰기 일관성 수정` |
| `docs` | 문서 수정 | `docs: 설치 가이드 업데이트` |
| `test` | 테스트 추가/수정 | `test: 로그인 테스트 케이스 추가` |

### 예시

```
feat: 사용자 인증 페이지 구현

fix: 폼 유효성 검사 오류 수정

refactor: 공통 컴포넌트 최적화

docs: README 한국어 번역 완료
```

## 특징

✨ **대화형 모드**
- 변경된 파일을 시각적으로 표시
- 각 커밋 단계에서 대화형으로 파일 선택

🌍 **한글 커밋메세지**
- 모든 커밋메세지는 한글로 작성 가능
- 타입은 영문으로 유지하여 일관성 보장

📝 **다양한 모드**
- 대화형 모드 (기본, 권장)
- 모든 변경사항 한 번에 커밋 (`--all`)
- 스테이징된 파일만 커밋 (`--staged`)

🎨 **컬러 출력**
- 정보는 `CYAN`
- 성공은 `GREEN`
- 경고는 `YELLOW`
- 지시사항은 `BLUE`
- 오류는 `RED`

## 주의사항

- 커밋메세지는 **필수** 입니다
- 파일 선택은 쉼표로 구분합니다 (예: `1,2,3`)
- `all`을 입력하면 모든 변경사항을 스테이징합니다
- 숫자는 1부터 시작합니다

## 트러블슈팅

### 커맨드를 찾을 수 없습니다

상대경로로 실행하세요:

```bash
./.claude/commands/git-commit
```

또는 프로젝트 루트에서:

```bash
bash ./.claude/commands/git-commit
```

### 선택이 적용되지 않습니다

숫자를 쉼표로 구분했는지 확인하세요:

```bash
# ✅ 올바른 형식
1,2,3

# ❌ 잘못된 형식
1 2 3
1-2-3
```

## 참고

이 도구는 다음과 같은 상황에 유용합니다:

- 여러 기능이 섞여 있는 변경사항을 분리할 때
- 커밋 히스토리를 깔끔하게 유지하고 싶을 때
- 기능별로 리뷰하기 쉽게 하고 싶을 때
- 실수로 관련 없는 파일을 함께 커밋하는 것을 방지하고 싶을 때
